We need to have a way to create a generic well known interface name to identify
which interface is for connection to the remarkable2.  I use a custom udev rule
based on the USB-C ethernet interface that is created when the rm2 is plugged in.

After plugging in the rm2, find the hardware identifier (idVendor and idProduct)
numbers. With a custom udev rule, we can create network interface usbnet0

ip link

3: ens192u4u1c2: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UNKNOWN mode DEFAULT group default qlen 1000
    link/ether 7a:e1:d3:58:43:55 brd ff:ff:ff:ff:ff:ff

udevadm info -a -p /sys/class/net/ens192u4u1c2

Scroll back and look for parent device attributes that have idVendor and idProduct, capture those values.

# setup udev rules to create a friendly network interface name
vi /etc/udev/rules.d/70-my-usb-net.rules 
SUBSYSTEM=="net", ACTION=="add", ATTRS{idVendor}=="04b3", ATTRS{idProduct}=="4010", NAME="usbnet0"

udevadm control --reload-rules

udevadm trigger

ip link

3: usbnet0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UNKNOWN mode DEFAULT group default qlen 1000
    link/ether 7a:e1:d3:58:43:55 brd ff:ff:ff:ff:ff:ff
    altname enp10s0u4u1c2
    altname ens192u4u1c2
    altname enx7ae1d3584355

Now we can setup a static IP address on that interface via NetworkManager

nmcli connection add type ethernet con-name usbnet-conn ifname usbnet0 ipv4.method manual ipv4.addresses 10.11.99.14/24
nmcli connection up usbnet-conn
ping 10.11.99.1
